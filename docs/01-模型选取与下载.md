# 01 · 模型选取与下载

部署的第一步不是配置环境，而是先确定要跑什么模型、从哪里把它完整地下载到本地——后续 Dockerfile 与量化方案的设计都取决于这一步的结论。术语可参考 [00-前置知识](00-前置知识.md) 的「三、核心术语卡」。

---

## 一、去哪找模型（来源）

| 来源 | 说明 | 适合 |
| :--- | :--- | :--- |
| **HuggingFace Hub**（hf.co） | 全球最大开源模型社区，Qwen / DeepSeek / Llama 等都在这里发布 | 最全，首选 |
| **ModelScope 魔搭**（modelscope.cn） | 国内平台，下载快、中文友好 | 国内网络、中文模型 |
| **官方发布渠道** | 模型官网 / 官方仓库，通常第一时间放权重 | 追新版本 |
| **镜像 / 加速源** | 清华、阿里等提供的 HuggingFace 镜像（如 `hf-mirror.com`） | 解决国内下载慢 |

> **门控（gated）模型**：部分模型发布方会对权重开启访问限制，通常出于许可证合规、安全审查或分批开放的需要。在 HuggingFace 上表现为仓库页面的 "Agree and access repository" 授权按钮。开启门控后，必须登录账号、接受协议（部分还需申请审批）并携带访问令牌（token）才能拉取**完整且正常的权重**；未授权直接下载拿到的文件往往缺失或内容损坏。

## 二、怎么判断"这个模型我跑得动"

按下面顺序快速筛一遍，避免下完才发现跑不了：

1. **参数量 vs 你的显存**：7B-FP16 ≈ 14GB，14B-FP16 ≈ 28GB，70B 必须多卡。先量显存再选。
2. **精度要求**：想用 FP8 省显存，显卡算力必须 ≥ 8.9（L40S / H100）；老卡（如 A100=8.0）会被迫反量化、显存翻倍（见 [04-模型量化](04-模型量化.md)）。
3. **是否门控**：有授权按钮 = 门控，必须带 token 下；否则用镜像"绕开授权"会拿到**大小正常但内容损坏的权重**，表现就是输出乱码。
4. **任务类型**：聊天用 chat 模型，检索用 embedding，画图用扩散模型（见 [12-ComfyUI 部署](12-ComfyUI部署.md)）。

> 🔧 **踩坑**：某 14B 推理模型用镜像无 token 下载 27.5GB 权重，加载成功但输出全是阿拉伯语乱码。带 HF Token 重下后正常中文输出。**选模型前先确认是否门控**：能直接看到文件列表 = 非门控；有授权按钮 = 门控。

## 三、选型经验法则

| 场景 | 建议 | 理由 |
| :--- | :--- | :--- |
| 入门练手 | 4B 左右小模型（如 Phi 系列） | 单卡就能跑，先跑通流程比追参数重要 |
| 通用对话 | 7B～14B 主流开源模型 | 性价比最高、资料最多 |
| 专业任务 | 看官方 benchmark + 社区评测 | 别只看参数量 |

## 四、下载方式（标准做法）

### 方式 A：huggingface-cli（最简洁，支持断点续传）

```bash
# 登录（门控模型需要）
huggingface-cli login --token hf_xxxx

# 下载到指定目录
huggingface-cli download meta-llama/Llama-3.1-8B-Instruct \
    --local-dir ./models/llama3 \
    --resume-download
```

### 方式 B：逐文件下载（网络不稳首选，可独立重试）

```python
from huggingface_hub import list_repo_files, hf_hub_download
import time

files = list_repo_files(repo_id)
for f in files:
    for attempt in range(30):
        try:
            hf_hub_download(repo_id, filename=f, local_dir='./model-path',
                          resume_download=True)
            break
        except Exception as e:
            time.sleep(2 ** min(attempt, 10))  # 指数退避，上限 10 分钟
```

### 方式 C：curl 断点续传（大文件最可靠）

```bash
curl -L -o model.safetensors -C - --retry 10 --max-time 600 <url>
```

> 🔧 **PowerShell 下载大文件注意**：`WebClient.DownloadFile()` 对 >5GB 文件无进度且超时挂死。用 `curl.exe`（不是 `curl` alias）：`curl.exe -L -o <file> -C - --max-time 600 <url>`。

### 镜像 / 代理加速

```python
import os
os.environ['HF_ENDPOINT'] = 'https://hf-mirror.com'   # 国内首选
os.environ['HF_HUB_DISABLE_SSL_VERIFY'] = '1'
```

> ⚠️ **只设这两个就够**，不要加 `urllib3`/`requests` 的复杂 SSL monkey-patch，反而引入 ImportError。

## 五、常见坑（🔧）

| 现象 | 原因 | 方案 |
| :--- | :--- | :--- |
| 下载静默退出（exit 0）但文件不全 | XetHub CAS bridge 新式下载不支持 | 改用「方式 B」逐文件 + 重试 |
| `ReadTimeout` / `403` | 镜像预签名 URL 过期（1h） | 逐文件下载 + 重试，网速慢时打碎文件 |
| 模型加载成功但输出乱码 | 门控模型无 token 下载到垃圾数据 | 带 HF Token 重新下载 |
| `PermissionError` 重跑失败 | 残留 `.lock` / `.incomplete` 文件 | 清缓存再重跑 |
| 磁盘空间不足 | 没预估体积 | 下载前用 `HfApi().model_info()` 查体积 |
| 🔧 **Windows 设 `HF_HUB_ENABLE_HF_TRANSFER=1` 报 SSL 异常** | Windows 无 Rust 编译的 `hf_transfer` 二进制 | **Windows 不用此环境变量**，Windows 上直接设 `HF_ENDPOINT` + 关闭 SSL 验证即可 |
| 🔧 **下载脚本静默崩溃无日志** | 主函数未加 try/except 兜底 | 主函数外加 `try: main() except: log(traceback.format_exc())`，否则什么日志都看不到 |

> 🔧 所有坑已在通用下载脚本里踩完——SSL 绕过方式、逐文件 vs snapshot、重试次数与退避、缓存清理、日志格式。新项目复用已验证脚本，只改 `REPO_ID` / `TARGET` / `LOG_FILE` 三个变量即可。

## 六、进阶速查（大表沉底）

### 下载源对比

| 源 | 速度 | 说明 |
| :--- | :---: | :--- |
| HuggingFace 直连 | 慢 | 部分地区被墙，生产不建议 |
| **HF-Mirror** | ⚡快 | 国内首选，CDN 加速 |
| ModelScope | 快 | 路径有嵌套陷阱（见下） |

> ⚠️ **ModelScope 路径陷阱**：`snapshot_download(cache_dir=...)` 会在目标目录下多创建一层 `组名/仓库名/` 嵌套，导致代码找不到模型；失败还会残留 `.cache/huggingface/download/` 下的 `.incomplete` 文件。**建议清缓存后用 hf-mirror 重下**，两者不要混用。

### 版本锁定（revision pinning）

```python
# ❌ 不锁定——作者更新权重后静默变更
model = AutoModel.from_pretrained("org/model")
# ✅ 锁定 commit hash（完整 40 位）或 tag
model = AutoModel.from_pretrained("org/model", revision="4e31d7a622b")
```

> ⚠️ `revision="main"` 并不能锁定——main 随时可能被覆盖。发布 tag 或完整 commit hash 才是可靠方式。

### 文件完整性校验

大文件下载后可能损坏（分片不完整），导致加载时 `key not found` 或输出异常。检查三步走：① 对比文件大小是否接近 HF 页面显示值；② 用 `safetensors.safe_open()` 打开测试；③ 加载时 `try/except` 捕获异常。详见 [11-文件完整性校验](11-文件完整性校验.md)。

### 门控模型处理

打开 HF 页面 → 能看到文件列表 = 非门控 ✅；有授权按钮 = 门控 ❌。门控模型带 HF Token 下载：

```python
import requests
headers = {'Authorization': f'Bearer {HF_TOKEN}'}
resp = requests.get(url, headers=headers, stream=True, verify=False)
```
