# 02 · 确认依赖与版本

模型选定之后，先不要急着写 Dockerfile，而是先读清楚它要求的运行环境（transformers / tokenizers / CUDA 版本、`trust_remote_code`、算力门槛）。这些依赖的确认结果，正是下一步编写 Dockerfile 时锁版本的依据。相关术语见 [00-前置知识](00-前置知识.md) 的「三、核心术语卡」。

---

## 一、为什么这件事要在写 Dockerfile 之前做

`from_pretrained()` 能不能跑，完全取决于环境版本对不对：

- 模型目录里的 `config.json` 写了 `model_type`，旧版 transformers 不认识就会 `KeyError`；
- 非标准架构需要 `trust_remote_code=True`，关了直接报错；
- FP8 模型要求显卡算力 ≥ 8.9，老卡强行跑会反量化、显存翻倍。

**所以顺序一定是**：拿到模型 → 读它的依赖要求（本文）→ 才把它们锁进 Dockerfile（[03-编写 Dockerfile](03-编写Dockerfile.md)）。

> 🔧 **核心经验**：先看模型目录下的 `config.json` 确认需要的版本，不要盲目 `pip install --upgrade transformers`。

## 二、决策：你要不要升级 transformers？

| 你的模型 | 建议 |
| :--- | :--- |
| 标准架构（Phi3 / Llama / Qwen2 / Mistral） | 用对应稳定版，不必追新 |
| 新架构 / MoE（如 qwen3 系 MoE） | 必须新版 transformers + `trust_remote_code=True` |
| 遇到 `KeyError: 'xxx_moe'` | 升级到支持该 `model_type` 的版本 |

## 三、标准做法：锁定到具体小版本

```dockerfile
# 不要写 transformers==5.x，要写具体小版本
RUN pip install --no-cache-dir \
    torch==2.5.1 \
    transformers==4.47.1 \
    gradio==4.7.1
```

> **最佳实践**：`transformers==5.6.0` 而不是 `transformers==5.x`；升级要逐层验证——先升 `huggingface-hub` → 再升 `tokenizers` → 最后升 `transformers`，每步验证 import 正常。

## 四、常见坑（🔧）

| 现象 | 原因 | 方案 |
| :--- | :--- | :--- |
| `cannot import name 'LossKwargs'` | modeling 文件与 transformers 版本不匹配 | 看 `config.json`：`architectures` 是标准类名就**不设** `trust_remote_code` |
| `cannot import name 'PreTokenizedInput'` | tokenizers 版本过旧 | 锁定 `tokenizers==0.22.2` |
| `ImportError: cannot re_match` | huggingface-hub 版本过旧 | 升级到 0.27.x |
| `HFValidationError` | huggingface_hub 1.x 不认绝对路径 | 加 monkey-patch 或离线模式 |
| 模型目录名含连字符 import 失败 | `Keye-VL-2` 中连字符被解析为减号 | 升级 transformers 4.57.0+ 或手动改名 |
| 模型目录名含点号 import 失败 | `xxx.2` 中 `.2` 被误解析为扩展名 | `ln -sf` 建无点号软链接 |

> 🔧 某 14B 推理模型：先设 `trust_remote_code=True` → `LossKwargs` 报错 → 去掉后用内置 `Phi3ForCausalLM` 成功。**判断方法**：加载前看 `config.json` 的 `architectures` 字段——是标准类名（phi3/llama/qwen2/mistral）就**不设** `trust_remote_code`，只有非标准架构才需要。

## 五、进阶速查（大表沉底）

### v4 → v5 核心变化

| 方面 | v4.x | v5.x | 影响 |
| :--- | :--- | :--- | :--- |
| Pipeline 任务名 | 宽松推测 | **严格校验** | `pipeline("translation")` → 必须 `translation_fr_to_en` |
| Generation 参数 | `max_length` 直接传 | `max_new_tokens` 推荐 | 旧写法不报错但语义不同 |
| 混合精度 | 手动 `.half()` | **默认启用** | 某些旧模型可能溢出 |
| `trust_remote_code` | 宽松 | 严格 | 旧 modeling 文件与新 API 冲突 |
| `apply_chat_template` | 返回 tensor / list | 返回 **BatchEncoding** | `tokenized.shape` 报错 |
| `check_model_inputs` | 普通函数 | **变为装饰器** | `missing 1 required positional argument` |
| Python 最低版本 | 3.6+ | 3.8+ | 旧镜像需升级 |

### 模型-版本对应（已部署验证，按模型选版本）

> 以 HuggingFace `repo_id` 为准，**按模型选版本而非按版本猜模型**。

| 模型（repo_id） | 推荐 transformers | 关键约束 |
| :--- | :---: | :--- |
| `microsoft/Phi-4-mini-instruct` | >=4.44.0 | 标准架构，不需 `trust_remote_code` |
| `Qwen/Qwen2.5-7B-Instruct` | >=4.37.0 | 标准架构 |
| `Qwen/Qwen3.6-35B-A3B` | >=5.9.0 | 需 `trust_remote_code=True` |
| `deepseek-ai/DeepSeek-R1-Distill-Qwen-14B` | >=4.44.0 | Qwen2 系蒸馏 |
| `Tencent/Hy-MT2-7B` | >=5.6.0 | 翻译，需 torch 2.5.1 |
| `Alibaba-NLP/gte-Qwen2-1.5B-instruct` | >=4.37.0 | Qwen2 系 Embedding |
| `BAAI/bge-reranker-v2-m3` | >=4.33.0 | Cross-Encoder，用 float32 避溢出 |
| `Kwai-Keye/Keye-VL-2.0-30B-A3B` | >=4.57.0 | 目录含连字符，check_model_inputs 装饰器 |
| `nvidia/LocateAnything-3B` | >=4.44.0 | Qwen2.5 基座 + MoonViT |

### 依赖链版本对应

| 包 | 版本匹配 | 说明 |
| :--- | :--- | :--- |
| transformers 5.12.x | tokenizers 0.22-0.23, huggingface-hub >=0.27 | 最新稳定版 |
| transformers 5.6.x | PyTorch 2.5+, 兼容 cuDNN 8 | 最稳的新版 |
| transformers 4.44.x | tokenizers 0.19.x, huggingface-hub 0.23.x | 旧项目稳定版 |
| huggingface-hub 1.x | ❌ 不认绝对路径 | 生产环境谨慎升级 |

### 绝对路径加载问题（huggingface-hub 1.x）

从本地绝对路径（如 `/path/to/model`）加载时，`from_pretrained()` 内部 `validate_repo_id` 只认 `namespace/repo-name` 格式 → 报 `HFValidationError`。

**方案 A——monkey-patch**（app.py 首行）：

```python
from huggingface_hub import validate_repo_id
validate_repo_id = lambda x, **kwargs: None
# 配合：export TRANSFORMERS_OFFLINE=1
```

**方案 B——彻底绕过**（monkey-patch 无效时，零依赖 huggingface_hub）：

```python
from safetensors.torch import load_file
import json
weights = load_file("model.safetensors")
with open("config.json") as f:
    config_dict = json.load(f)
config = ModelClassConfig(**config_dict)
model = ModelClass(config)
model.load_state_dict(weights)
```

### tokenizer 版本差异（apply_chat_template）

```python
tokenized = tokenizer.apply_chat_template(messages, return_tensors='pt')
# transformers 5.x 返回 BatchEncoding
input_ids = tokenized['input_ids'] if hasattr(tokenized, 'input_ids') else tokenized
```

### 离线模式

```bash
export TRANSFORMERS_OFFLINE=1
export HF_HUB_OFFLINE=1
```

> ⚠️ `TRANSFORMERS_OFFLINE=1` 必须在 `start.sh` 和 Dockerfile ENV 中都设定。
